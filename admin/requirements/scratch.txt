Once users go on the webpage, they'll be welcomed with the home screen which lists all the blood donation & request posts. To be able to create a post, they are required to create an account by filling in the necessary details which are name, username, email, password, phone_number. Afterwards, they may login and start creating posts.

# request posts
- if a user selects one of the confirmed diseases, the post automatically gets rejected
- if a user selects "other" and adds in a different disease name, the post is marked as pending and an admin has to approve it
- if a user doesn't select any diseases, the post gets listed automatically

# donation posts
- if a user selects one of the confirmed diseases, the post automatically gets rejected
- if a user selects "other" and adds in a different disease name, the post is marked as pending and an admin has to approve it
- if a user doesn't select any diseases, the post gets listed automatically

- whole blood donations:  1 donation post every 56 days 
- power red donations:    1 donation post every 112 days
- platelet apheresis:     1 donation post every 7 days
- show a colour or message on each post to show if it's compatible with the user or not
- users between 18 and 60 years of age can donate


# manage users
- donors -> lists all the donors (SELECT * FROM users WHERE user_id IN (SELECT user_id FROM posts WHERE post_type = 'donate');
- supplicants -> lists all the supplicants (SELECT * FROM users WHERE user_id IN (SELECT user_id FROM posts WHERE post_type = 'request');

# manage history
- donations -> lists all donations done by all users (select * from posts where post_type = "donation")
- requests -> lists all requests done by all users (select * from posts where post_type = "request")
